# 沙箱性能对比报告：进程级 vs. Docker 容器级

**日期**: 2026-02-25
**作者**: Manus AI

---

## 1. 核心结论 (Executive Summary)

本次基准测试旨在量化**进程级沙箱**（优化前）与 **Docker 容器级沙箱**（优化后）之间的性能差异。测试结果清晰地揭示了一个核心权衡：**为了换取生产级的安全性、隔离性和环境一致性，Docker 容器化引入了可量化的性能开销**。

**关键发现：**

*   **性能开销**: Docker 沙箱在各项操作上均表现出更高的延迟。**冷启动延迟**是进程级沙箱的 **~97倍**（184.5ms vs 1.9ms），**简单命令执行**延迟是其 **~60倍**（103.2ms vs 1.7ms）。对于计算密集型任务（如 NumPy），额外开销相对较小（~3.1倍）。
*   **吞吐量下降**: Docker 沙箱的命令执行吞吐量显著低于进程级沙箱（**11.6 cmd/s** vs **600.8 cmd/s**），相差约 **52倍**。这主要是由于 Docker API 调用和容器内执行的固定开销。
*   **安全性与功能巨大提升**: 尽管性能下降，但 Docker 提供了**进程/文件系统/网络隔离**、**资源限制（CPU/内存）**、**环境一致性**和**休眠/唤醒**等关键能力，这些是进程级沙箱完全不具备的。从雷达图上看，Docker 沙箱在所有安全维度上的得分远超进程级沙箱。
*   **内存优势**: 单个 Docker 容器的基线内存占用极低（**~0.35 MB**），远低于进程级沙箱为执行任务而产生的子进程峰值内存（**~41.53 MB**），展示了更高效的静态资源管理。

**综合评估**：对于需要安全、可靠、可复现地执行任意代码的 AI Agent 系统而言，**Docker 容器化带来的安全性和功能性收益，远超其性能成本**。这种开销在用户感知的交互层面通常是可接受的（大部分操作仍在 100-200ms 量级），是迈向生产级系统的必要步骤。

## 2. 性能分析与数据可视化

### 2.1 操作延迟对比

下图展示了两种沙箱在各项基准测试中的平均操作延迟。请注意，Y 轴为**对数尺度**，以清晰展示数量级差异。

![操作延迟对比](chart_latency_comparison.png)

**分析**:
*   **启动延迟**: Docker 的冷启动（创建新容器）和热启动（在已运行容器中执行）延迟显著高于进程级沙箱。这是 Docker 容器化最主要的开销来源。
*   **命令执行**: 无论是简单命令还是复杂管道命令，Docker 的执行延迟都稳定在 100ms 左右，而进程级沙箱则在 10ms 以内。
*   **文件 I/O**: Docker 通过 Volume 挂载进行文件操作，其延迟同样高于直接的文件系统调用。
*   **并发会话**: 并发创建 5 个 Docker 容器耗时近 1 秒，而创建 5 个进程几乎是瞬时的。这体现了容器创建的资源密集性。

### 2.2 额外开销倍数

为了更直观地理解性能成本，下图展示了 Docker 沙箱相对于进程级沙箱的延迟倍数。

![额外开销倍数](chart_overhead_ratio.png)

**分析**:
*   **高开销区（橙色）**: 启动、简单命令、文件读取和并发会话创建的开销倍数最高（30x 到 190x）。这些操作本身非常快，因此 Docker 的固定开销显得尤为突出。
*   **低开销区（蓝色）**: 对于本身耗时较长的计算密集型任务（如 Python NumPy），Docker 的额外开销占总时长的比例较低，倍数也相应降低（3x 到 8x）。这表明对于复杂任务，性能差异不那么悬殊。

### 2.3 吞吐量与内存占用

![吞吐量与内存占用](chart_throughput_memory.png)

**分析**:
*   **吞吐量**: 进程级沙箱的吞吐量极高，因为它仅仅是创建新进程。Docker 沙箱的吞吐量受限于 Docker Daemon 的处理能力和 API 调用开销，但对于 AI Agent 的典型应用场景（一次执行一个工具调用）而言，11.6 cmd/s 的吞吐量已足够。
*   **内存占用**: Docker 容器在空闲状态下几乎不消耗内存（0.35 MB），而进程级沙箱在执行任务时会产生数十 MB 的子进程内存峰值。这表明 Docker 在大规模部署多会话时，静态内存管理更具优势。

## 3. 安全性与功能对比

### 3.1 安全性 vs. 性能综合评估

雷达图从 8 个维度全面评估了两种方案。**面积越大，代表综合能力越强**。

![安全性 vs. 性能雷达图](chart_radar_comparison.png)

**分析**:
*   **进程级沙箱（绿色）**: 在**性能相关**的四个维度（执行速度、启动速度、文件 I/O、并发能力）上得分接近满分，但在**安全相关**的四个维度上得分极低。
*   **Docker 容器沙箱（蓝色）**: 在**安全相关**的四个维度（进程隔离、文件系统隔离、资源限制、环境一致性）上得分接近满分，但在性能维度上有所妥协。

**结论**: Docker 沙箱以适度的性能牺牲，换来了体系化的安全保障，整体能力模型更为均衡和强大，符合生产环境要求。

### 3.2 Docker 独有能力：生命周期管理

Docker 容器化带来了全新的生命周期管理能力，这是进程级沙箱无法实现的。

![Docker 生命周期操作延迟](chart_docker_lifecycle.png)

**分析**:
*   **休眠唤醒**: Docker 容器可以在空闲时被 `stop`（休眠），并在需要时被 `start`（唤醒）。唤醒操作的延迟（~120ms）远低于冷启动（~185ms），这使得系统可以在节省资源的同时，快速响应用户请求。这是对标 Manus 1.6 Max VM 快照恢复的关键功能实现。

## 4. 基准测试方法

*   **测试环境**: Ubuntu 22.04, Docker 26.1.4, Python 3.11
*   **测试脚本**: `tests/benchmark_sandbox.py`
*   **测试指标**:
    *   **启动延迟**: 创建沙箱环境所需时间。
    *   **命令执行延迟**: 执行简单、复杂、Python 脚本的时间。
    *   **文件 I/O**: 读写 1MB 文件和创建 100 个小文件的时间。
    *   **并发能力**: 并发创建 5 个独立会话所需时间。
    *   **吞吐量**: 5 秒内可执行的简单命令总数。
    *   **内存占用**: 任务执行过程中的峰值内存使用。
*   **统计方法**: 每项测试预热 1 次，正式运行 5 次，结果取平均值。
