# Docker 沙箱监控告警系统 — 优化报告

## 一、概述

本次优化为 Manus MVP 项目的 Docker 沙箱添加了完整的 **容器资源监控** 和 **异常告警** 系统，包含后端监控采集引擎、RESTful API、WebSocket 实时推送，以及前端可视化仪表盘界面。整个系统采用零侵入式设计，对原有代码的修改仅限于在 `main.py` 中注册监控路由（3 行代码）和在 `App.tsx` 中添加路由（2 行代码）。

### 2.2 数据流

监控系统的数据流遵循 **采集 → 存储 → 检测 → 推送** 的管道模式。采集循环每 5 秒执行一次，通过异步子进程调用 `docker ps` 和 `docker stats` 获取容器资源数据。采集到的原始数据经过解析后存入内存时间序列（每容器最多保留 720 个数据点，即 1 小时），同时触发告警引擎的阈值检测。检测结果通过 WebSocket 实时推送至所有已连接的前端客户端。

## 四、后端监控引擎详解

### 4.1 数据采集

采集模块通过 `asyncio.create_subprocess_exec` 异步调用 Docker CLI 命令，避免阻塞事件循环。采集的指标包括：

| 指标 | 来源命令 | 说明 |
|:-----|:---------|:-----|
| CPU 使用率 | `docker stats --no-stream` | 百分比，精确到小数点后两位 |
| 内存使用量/限制 | `docker stats --no-stream` | 字节级精度，同时计算百分比 |
| 网络 I/O 速率 | `docker stats --no-stream` | 通过前后两次采集差值计算 bytes/s |
| 磁盘 I/O 速率 | `docker stats --no-stream` | 通过前后两次采集差值计算 bytes/s |
| 进程数 | `docker stats --no-stream` | 容器内 PID 数量 |
| 容器状态 | `docker ps -a` | Up/Exited/Created 等 |

### 4.2 告警引擎

告警引擎采用 **双阈值** 设计（Warning / Critical），支持 4 种预设指标的阈值检测：

| 指标 | Warning 默认值 | Critical 默认值 | 说明 |
|:-----|:--------------:|:---------------:|:-----|
| `cpu_percent` | 70% | 90% | CPU 使用率 |
| `memory_percent` | 75% | 90% | 内存使用百分比 |
| `memory_usage_mb` | 384 MB | 480 MB | 内存绝对使用量 |
| `pids` | 192 | 240 | 进程数量 |

告警引擎具备以下特性：

**冷却机制**：同一容器的同一指标在 30 秒内不会重复触发告警，避免告警风暴。

**自动恢复**：当指标回落到阈值以下时，告警自动标记为已解决（resolved），并记录恢复时间。

**历史保留**：最多保留 1000 条告警历史记录，支持按容器名称和告警级别过滤查询。

### 4.3 WebSocket 实时推送

监控引擎内置 WebSocket 广播器，每次采集完成后自动向所有订阅者推送两种类型的消息：

```json
// 类型 1: metrics_update — 包含所有容器的最新指标
{
  "type": "metrics_update",
  "data": { "overview": {...}, "containers": {...} }
}

// 类型 2: alert — 新产生的告警事件
{
  "type": "alert",
  "data": { "container_name": "...", "metric": "cpu_percent", "level": "critical", ... }
}
```

## 五、后端 API 端点

| 方法 | 路径 | 说明 |
|:-----|:-----|:-----|
| `GET` | `/api/monitor/overview` | 系统总览（容器数、总 CPU/内存、告警数） |
| `GET` | `/api/monitor/containers` | 所有容器的当前指标 |
| `GET` | `/api/monitor/containers/{name}` | 单个容器的当前指标 |
| `GET` | `/api/monitor/history/{name}` | 容器历史时间序列（支持 `duration` 参数） |
| `GET` | `/api/monitor/alerts` | 告警历史（支持 `container_name` 和 `level` 过滤） |
| `GET` | `/api/monitor/alerts/active` | 当前活跃告警 |
| `GET` | `/api/monitor/thresholds` | 获取告警阈值配置 |
| `PUT` | `/api/monitor/thresholds/{metric}` | 更新告警阈值 |
| `WebSocket` | `/ws/monitor` | 实时监控数据推送 |

## 六、前端仪表盘功能

### 6.1 总览区域

页面顶部展示 6 张总览卡片，分别显示容器总数（运行/停止）、总 CPU 使用率、总内存使用量（含限制）、网络 I/O 速率、活跃告警数量、监控引擎运行时间。每张卡片使用毛玻璃效果（`backdrop-blur`）和渐变色图标，视觉风格与项目整体暗色主题一致。

### 6.2 系统趋势图

总览下方展示三组迷你趋势图（CPU、内存、网络），使用 SVG 折线图实现，实时更新，最多显示最近 30 个数据点。图表区域使用渐变填充增强可读性。

### 6.3 容器列表

左侧展示所有容器的卡片列表，每张卡片包含：

- 容器名称和运行状态（绿色/灰色指示灯）
- CPU 使用率进度条（带百分比标签）
- 内存使用量（MB 和百分比）
- CPU 和内存的迷你趋势图
- 可展开的详细面板（点击展开按钮查看更多指标）

### 6.4 告警中心

右侧展示告警面板，包含两个标签页：

**活跃告警**：显示当前未解决的告警，每条告警包含容器名称、指标名称、当前值、阈值、告警级别（warning 黄色 / critical 红色）、触发时间。

**历史记录**：显示所有告警历史，已解决的告警会显示恢复时间。支持一键清除历史记录。

### 6.5 告警阈值配置

右下方展示当前的告警阈值配置，每个指标显示 Warning 和 Critical 两个阈值，支持点击"编辑"按钮弹出内联编辑器，修改后通过 API 实时生效。

### 6.6 实时更新机制

前端通过 WebSocket 连接 `/ws/monitor` 端点，接收实时监控数据。连接状态在页面右上角显示（已连接/已断开/连接中），支持自动重连（3 秒间隔）。用户可通过"暂停"按钮临时停止数据更新，或通过"刷新"按钮手动触发一次 REST API 拉取。

## 七、技术亮点

### 7.1 零侵入集成

整个监控系统通过 3 个新文件实现，对原有代码的修改仅 8 行。监控 API 的注册使用 `try/except` 保护，即使监控模块不存在也不影响主服务启动。

### 7.2 异步非阻塞采集

所有 Docker CLI 调用均通过 `asyncio.create_subprocess_exec` 异步执行，不会阻塞 FastAPI 的事件循环。每次采集的 `docker stats` 调用耗时约 1-2 秒，但不影响 API 响应。

### 7.3 内存高效的时间序列

使用 Python `deque(maxlen=720)` 实现固定长度的环形缓冲区，每个容器最多保留 720 个数据点（5 秒间隔 × 720 = 1 小时），自动淘汰旧数据，无需额外的垃圾回收。

### 7.4 Docker Socket 权限自适应

监控引擎在启动时自动检测 Docker socket 的访问权限。如果当前用户无权访问 `/var/run/docker.sock`，会在日志中给出明确的提示信息，而不是静默失败。

### 7.5 前端动画与交互

前端使用 Framer Motion 实现容器卡片的布局动画（添加/移除容器时平滑过渡），告警弹出时带有淡入动画，数值变化时带有 CSS transition 过渡效果。

## 八、使用指南

### 8.1 访问仪表盘

启动服务后，访问 `http://localhost:3000/monitor` 即可打开监控仪表盘。也可以从主页面点击导航链接进入。

### 8.2 确保 Docker Socket 可访问

监控引擎需要访问 Docker socket 来采集容器数据。如果后端进程的用户不在 `docker` 组中，需要执行：

```bash
# 方法 1: 将用户加入 docker 组
sudo usermod -aG docker $USER

# 方法 2: 临时开放 socket 权限
sudo chmod 666 /var/run/docker.sock
```

### 8.3 自定义告警阈值

通过 API 或前端界面均可修改告警阈值：

```bash
# 通过 API 修改 CPU 告警阈值
curl -X PUT "http://localhost:8000/api/monitor/thresholds/cpu_percent?warning=60&critical=85"
```
